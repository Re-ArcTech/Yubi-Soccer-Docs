### 3.2 進捗報告

#### カシュー

| | |
|---|---|
| **進捗①** | MediaPipe プロジェクトのパッケージ化 → ゆびサッカー本体プロジェクトへの取り込みに成功。想定より簡単に完了 |

#### りーせ

| | |
|---|---|
| **進捗①** | メッセージ型定義（前回MTGで決めたフォーマット）を `yubi-server` リポジトリに暫定アップロード。内容: メッセージの種類・連番・タイムスタンプ・ペイロード構造、各イベントのエンコード定義 |
| **進捗②** | 通信同期の調査。Unity + Go サーバーの簡易プロトタイプを構築し、プレイヤー同期の検証を実施中 |
| **結果** | クライアント予測 + サーバーロールバック（ハイブリッド型）が体感良好と確認 |
| **課題** | サーバーから訂正座標を即送信すると**カクつき**が発生する。座標の差分を滑らかにする**補間処理**（PositionInterpolation）が必要。更新間隔を一定にする方法も検討したが、スマホの通信環境を一定前提にするのはリスクがあるため、補間処理を入れる方針で進める |
| **来週** | ボールの同期調査（Rocket League 実装の参考コードを確認済み） |

### 3.4 ユビキタス言語定義の確認（エンジニア3名）

定例MTG後、カシュー・りーせ・ざきこうでユビキタス言語（共通用語定義ドキュメント）を確認・レビュー。

#### 確定した用語定義

Yubi-Soccer-Docsに投稿予定。

#### MatchState の追加分割

KickOff を以下の2状態に分割することで決定:

| 状態 | 内容 |
|---|---|
| **MatchStart 後のカウントダウン** | 試合開始時の 3・2・1 カウント → `Ready` ステートに含める |
| **KickOff** | ゴール後〜再開までの一連の処理（スコア反映・演出・カウントダウン含む） |

#### serverのメッセージ定義ドキュメント（`yubi-server` リポジトリ）

- まだ暫定版のため、このリポジトリ（Docs）には未反映
- バージョン互換性の仕組みは実装済み
- **確定次第 Docs に取り込み、整合性を確認する**
- 確認優先ファイル: `types.go` / `errors.go`

#### 未決・追加検討事項

- [ ] `PositionInterpolation` をクライアント側かサーバー側どちらで管理するか（サーバー外部注入の可能性あり）
- [ ] `BallOwner` / `BallOwnership`（ボール所有者）の定義追加

## リポジトリの役割

このリポジトリは **企画・議事録・タスク** のみを管理します。  
開発ルール・コード・Unity アセットは各実装リポジトリに置きます。

```
Yubi-Soccer-Docs/
├── AGENTS.md          ← このファイル
├── README.md
├── Proposal/          # 企画書
├── Meetings/          # 議事録（1会議 = 1ファイル）
└── Tasks/             # タスク一覧（ミーティングごとに更新）
```

## AI エージェントへの基本方針

**不明点・曖昧な点は必ず議事録作成者に全て確認してから作業を進めること。**  
推測で補完しない。特に以下は必ず確認する。

- 参加者・欠席者が不明な場合
- 発言者が特定できない場合
- 決定事項か未決事項か判断できない場合
- 日時・次回ミーティング日程が不明な場合

---

## 議事録の作成ルール

1. ファイル名: `Meetings/YYYY-MM-DD-（トピック）.md`
2. テンプレート: [`Meetings/_template.md`](./Meetings/_template.md) をコピーして使う
3. 各セクションの **現状 / 問題 / 選択肢 / アプローチ / 結果** を必ず埋める
4. 決定事項は `Tasks/` にも反映する
5. 個人名は上記辞書のニックネームのみ使用

## タスクの更新ルール

1. ファイル名: `Tasks/tasks-YYYY-MM-DD.md`（ミーティング日付）
2. ステータス記号: `🔲 未着手` / `🔄 進行中` / `✅ 完了` / `⏸ 保留`
3. `Tasks/README.md` の一覧テーブルも必ず更新する

## コミットルール

- 本名・個人情報をコミットに含めない
- コミットメッセージ形式: `docs: （内容の要約）`
- push 前に `git diff origin/main..HEAD` で本名漏れがないか確認する
